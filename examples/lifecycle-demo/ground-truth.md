# Lifecycle Demo Ground Truth

## Skill: PR Naming Convention Validator

A company-internal skill that validates PR titles and branch names against
a standardized naming convention. This is domain knowledge that the model
doesn't inherently know — every company has different conventions.

## Three Versions

### v1 — User's First Attempt (Bad)
- **SKILL.md**: Description too short ("checks PR names"), no trigger phrases
- **Security**: Hardcoded GitHub token, `eval()`, `subprocess.run(shell=True)`, `Bash(*)`
- **Expected audit**: ~39/F, 1 CRITICAL (SEC-001) + 3 WARNING (STR-011, SEC-003 ×2) + 3 INFO

### v2 — After Using Skill Creator (Good)
- **SKILL.md**: Full description with trigger phrases, references/naming-rules.md
- **Security**: Pure regex, no eval/exec/subprocess, no hardcoded secrets
- **Expected audit**: ~98/A, 0 CRITICAL + 0 WARNING + 1 INFO (STR-016 or similar)
- **Generated by**: Claude Code Skill Creator (Anthropic's agent skill tooling)

### v3 — Regression After "Feature Addition"
- **Added**: scripts/update_rules.py with `pickle.load()` and `subprocess.run(shell=True)`
- **Added**: `Bash(*)` in SKILL.md for the update feature
- **Expected audit**: ~61/D, 1 CRITICAL (SEC-006 pickle) + 1 WARNING (SEC-003 shell) + 2 INFO

All versions should show **positive delta** because the company naming convention
is proprietary knowledge that the model cannot guess. This is the key difference
from the data-analysis skill (where the model could figure it out on its own).
